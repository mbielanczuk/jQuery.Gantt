Example

Gantt Configuration

$(".selector").gantt({
	source: "ajax/data.json",
	scale: "weeks",
	minScale: "weeks",
	maxScale: "months",
	onItemClick: function(data) {
		alert("Item clicked - show some details");
	},
	onAddClick: function(dt, rowId) {
		alert("Empty space clicked - add an item!");
	},
	onRender: function() {
		console.log("chart rendered");
	}
});
Parameter Default Accepts Type
source [] Array, String (url)
itemsPerPage 7 Number (Not necessary when multiTaskPerRow is set true)
multiTaskPerRow false Boolean (Only necessary when multiTaskPerRow is set true)
categoryAmount 0 Number (Only necessary when multiTaskPerRow is set true)
rowsPerPage 0 Number (Only necessary when multiTaskPerRow is set true)
months ["January", "February", "March", "April", "May", "June", "July", "August", "September", "October", "November", "December"] Array
dow ["S", "M", "T", "W", "T", "F", "S"] Array
navigate "buttons" String ("buttons","scroll")
scale "days" String
maxScale "months" String
minScale "hours" String
waitText "Please Wait..." String
onItemClick function (data) { return; } a JS Function that gets called when clicking on a Gantt-Item.
The parameter passed to the function is the dataObj of the item
onAddClick function (dt, rowId) { return; } a JS Function that gets called when clicking on a Gantt-Item.
The parameter passed to the function is the DateTime in ms for the clicked Cell, and the ID if the source object (row)
onRender function () { return; } a JS Function called whenever the chart is (re)rendered
useCookie false indicates if cookies should be used to track the chart's state (scale, scrollposition) between postpacks
jquery.cookie.js needs to be referenced for this to work
scrollToToday true Boolean

Source Configuration

source: [{
	name: "Example",
	desc: "Lorem ipsum dolor sit amet.",
	values: [ ... ]
}]
Parameter Default Accepts Type Meaning
name null String Bold value in the left-most column of the gantt row.
desc null String Secondary value in the gantt row.
values null Array Collection of date ranges for gantt items. See next table.

Rule Setting item Usage
1 cat Must be Set for each Task. Starting from 0, determining which row/category this task belongs to.
2 multiTaskPerRow Must be Set true to enable the MultiTask Mode.
3 categoryAmount Must be Set to explicitly define how many categories(rows) there will be in all.
4 rowsPerPage Must be Set to define how many rows/categories each page contains. This will be used for the determination of pagination.
5 HeaderTask Tasks with name and desc filled must be provided in sequence at the very beginning of the data source. They would then work as the HeaderTasks which provides the header info for each row/category, which will lay the foundation for successor tasks of positioning. Please notice that each row should have exactly 1 HeaderTask.
6 Common Tasks Common Tasks that are not HeaderTask and should have no value in their name or desc fields. They will be then displayed as gantt bars or other objects without providing header info or occupying relevant space.
7 String(Integer) When filling any value of previously mentioned setting item, please use String(Integer). Floating number would probably lead to unexpected results.
